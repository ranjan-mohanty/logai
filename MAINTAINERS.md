## Maintainer Responsibilities

Maintainers are responsible for:

### Code Review

- Review and merge pull requests
- Ensure code quality and consistency
- Provide constructive feedback to contributors

### Issue Management

- Triage new issues
- Label and prioritize issues
- Close resolved or invalid issues
- Respond to community questions

### Release Management

- Plan and execute releases
- Update changelog and documentation
- Publish to crates.io
- Create GitHub releases

### Community Management

- Foster a welcoming community
- Enforce Code of Conduct
- Recognize contributors
- Engage with users

### Documentation

- Keep documentation up-to-date
- Review documentation PRs
- Improve clarity and completeness

### Security

- Respond to security reports
- Coordinate security fixes
- Publish security advisories

## Becoming a Maintainer

We welcome new maintainers! To become a maintainer:

1. **Contribute regularly** - Make consistent, quality contributions
2. **Show expertise** - Demonstrate deep understanding of the codebase
3. **Help others** - Review PRs, answer questions, help newcomers
4. **Be reliable** - Follow through on commitments
5. **Align with values** - Uphold project values and Code of Conduct

### Process

1. Existing maintainers nominate candidates
2. Candidate accepts nomination
3. Maintainers vote (majority approval required)
4. New maintainer added to team
